Agents Assemble

Building a command center for eighteen AI agents — and accidentally giving them personalities along the way.

I The Invisible Infrastructure March 17–18, 2026

By March 18th, I had eleven AI agents running out of my terminal. Each one specialized. Chiron tracked my health. Midas managed finances across three entities. Genos maintained a research paper database. Saber curated my taste profile. Sisyphus drafted my newsletter by coordinating four sub-agents in parallel.

They had enforcement policies, session logs, shared context files. An entire infrastructure humming along in ~/.claude/. Hooks that fired on every tool call. A JSONL log with 207 lifecycle events. All of it built between March 17th and 18th — two evenings of wiring hooks, writing enforcement profiles, and configuring session loggers.

But I couldn't see any of it. Checking on an agent meant grepping through log files. Verifying one was working meant reading its learnings directory by hand. I had no way to know, at a glance, which agents had been active that week, which had stale data, or whether the automation was actually running.

I needed a dashboard. Not a monitoring tool — something that felt like mine.

II Faces

Before the dashboard, the agents needed faces. They already had names drawn from mythology, anime, and literature — Chiron the centaur trainer, Borges the librarian, Saitama the overpowered hero who finds everything too easy. But names in a terminal only go so far.

The aesthetic was decided immediately: 1-bit pixel art. Black and white. No color, no gradients, no anti-aliasing. The constraint was the point. Like a Macintosh icon from 1984, each character had to be recognizable through pure composition alone.

Early ChatGPT attempt. Impressive detail, wrong vibe entirely.

I started with ChatGPT. The results were technically impressive and completely wrong. A Saitama came back with a yellow cape rendered in full color — missing the entire point of the constraint. The mythological figures looked like textbook illustrations. Good art, wrong project.

ChatGPT (color leak)
Final (strict 1-bit)

So I built a pipeline instead. Gemini generated base images at high resolution, then a Bun script quantized them to an 80x80 grid in strict 1-bit — no gray, no compromise — and scaled them up to 640x640 for crisp rendering. For each agent, the script ran parameter sweeps: 24 variants across dithering algorithms and thresholds. I'd scan the grid and pick the one with the right expression.

Saber parameter sweep. 24 variants. One gets picked.

William James was the hardest. The father of American psychology has a recognizable face — the beard, the penetrating gaze — but at 80x80 pixels in pure black and white, he kept coming out as a generic Victorian gentleman. Five versions before he read as distinctly him.

But the best story was Sisyphus.

His original image was this heavy-browed, thick-necked strongman. Looked like he chose to push the boulder. Fit, powerful, unbothered. The problem was that the bearded sage I'd generated for Chiron looked too ancient and wise for a health coach — so the two got swapped. The sage became Chiron. And suddenly Sisyphus needed a new face.

The prompt became: make him look like he's been pushing a boulder forever.

Each iteration made him more wretched. Gaunt, hollow-eyed, messy hair. Visible collarbones. The kind of face that's been doing the same thing for millennia and knows it will never end. The newsletter agent — the one that curates and drafts and ships every week, forever — deserved to look exactly this tired.

One rule emerged that I hadn't expected: canvas fill matters. Early avatars filled maybe 40% of the frame and looked lost on the cards. I set a guideline — 70-80% fill, heads near the top — and re-rendered the entire batch.

The converter tool. Dithering, threshold, contrast — all tunable.

By end of day, seventeen portraits. Each one recognizable at 28 pixels wide.

III The Build Saturday, March 21, 2026

Saturday evening, March 21st. I sat down knowing I wanted to build the dashboard but not exactly how. The first thing I did was have Claude audit the full agent infrastructure — every definition file, every hook, every log. The result: 11 active agents, 118+ tools, enforcement profiles, observability hooks, cross-agent context files. More complex than I'd realized.

Then I found something I'd forgotten about: a zip file in my Downloads called "Agent Dashboard.zip." A complete HTML mockup with simulated data and placeholder avatars. I don't remember making it. It might have been a Claude artifact from weeks earlier. But it was exactly the reference I needed.

The architecture was deliberately simple. A Bun server on port 3141. Zero dependencies — no React, no framework, no bundler. Vanilla JavaScript with 30-second polling. A hardcoded registry, because the canonical list should be explicit, not discovered from the filesystem. Departure Mono for the typography. Warm cream background, burnt orange accent. No blue. No corporate gray.

By 10:47 PM — roughly two hours after starting — I had four pages.

Overview. Agent cards with avatars, session stats, class badges, and a live activity feed.

The overview was the front page — every agent as a card with its pixel portrait, domain, session count, and class badge. Below it, a row of automations showing what runs on schedule, and a live activity feed tailing the JSONL session log. At a glance: who's active, who's idle, what just happened.

Topology. Who talks to whom, who shares tools, who depends on whose data.

The topology page mapped relationships I hadn't fully seen before. Data dependencies, shared tools, consultation lines. Sisyphus at the bottom, orchestrating four sub-agents. Saber off to the side, connected to almost nothing — the loner of the roster. Below the graph, a tool registry showing every script and which agents have access. Eighteen tools across the ecosystem, each with explicit ownership.

Stats. Session counts, durations, and a GitHub-style activity heatmap.

The stats page parsed every session from the JSONL log — pairing start and stop events, computing durations, flagging orphans where an agent started but never stopped (probably crashed). Sisyphus led with 11 sessions. Saber close behind with 10. Half the roster had never been formally invoked yet — they existed as definitions waiting for their first real task.

Queues. A manually-curated kanban board — what each agent should work on next.

The queues page was the surprise. I'd originally planned to pull automated data — pending reviews, learnings counts, context file health. Useful but lifeless. I pivoted to a manually-maintained kanban: three columns, tasks I write myself, each assigned to a specific agent. A "Start" button that opens a terminal session with context pre-loaded. The dashboard went from something I watch to something I use to direct.

V Ranks

The tier labels — T1, T2, T3 — were clinical. These agents had faces and voices now. They needed a ranking system with more character.

I borrowed from anime: C Class, B Class, A Class, S Class.

C Class agents have a definition and nothing else. Manual invocation only. B Class have a skill, session modes, learnings — the core roster. A Class orchestrate other agents. Sisyphus spawns four sub-agents to produce the newsletter. Saitama coordinates parallel literature searches and runs quality gates on the results.

S Class remains empty. I'm not sure what earns it yet.

VI The Messenger

The last addition was the one that changed the relationship. A small paper plane icon in the bottom-right corner. Click it, and a panel slides open — threads grouped by agent, each one with their avatar and a short message.

The agents could now message me.

Not as chat. One-way notifications. Chiron nudges when I haven't logged a health check-in. Borges flags stale content in the vault. Midas surfaces new transactions waiting for categorization. A lightweight script runs every few hours, checks conditions, and sends reminders on behalf of agents — no full session required.

The deduplication was important: if an unread message with the same content already exists, it gets suppressed. No spam. Just one gentle reminder until acknowledged. Opening a thread marks everything as read. See the badge, read the message, get on with your day.

It's a small thing. But it's the difference between agents that wait for you to remember them and agents that reach out when something needs attention.

VII Vital Signs Sunday, March 22, 2026

The faces were static. Always the same expression, no matter what was happening. Sisyphus looked equally worn whether he was mid-publish-crunch or hadn't been invoked in three weeks. That felt wrong.

So I gave each agent five mood levels. L1 through L5. Five variations of each portrait — from depleted to thriving. The idea was simple: the face you see should tell you something about the agent's actual state.

Five moods per agent. L3 (orange) is baseline. Some follow the expected arc — Midas goes from neglected to thriving. Others don't: Sisyphus is inverted (destroyed = productive), Saitama barely changes, and Prometheus maps to his myth.

The range of approaches is the part I like most. Midas and William James follow the expected arc — neglected on the left, thriving on the right. Straightforward. But Saitama, the One Punch Man character who is bored by being too powerful, barely changes at all. Squint at L1 and L5 and you might notice a slightly different expression. That's the joke — he's imperceptible because he doesn't care.

Sisyphus is fully inverted. He's a newsletter agent. His whole identity is the endless push. When he's healthy and rested, it means nothing is shipping. When he looks like death — hair thinning, eyes hollow, boulder weighing him down — that's publish week. That's when he's doing his job. And Prometheus maps directly to his myth: L1 is the eagle attack, L5 is the blazing fire-bringer.

The signals that drive the moods are deliberately analog. No LLM calls. Just file dates, session counts, and inbox sizes. Chiron's mood is tied to how many health checkins exist in the past seven days. Borges reflects how cluttered the Obsidian inbox is. Midas dims when bank imports are overdue. The most digital system I've built runs on the most analog possible measurements.

Each mood also carries a reason and an optional action. Hover over Borges at L1 and you'll see: "Vault neglected — 23 inbox items, no triage in 14+ days" with a nudge to run /borges triage. It's not just decorative. The faces are a glanceable health check for the whole ecosystem.

The three-lane layout came at the same time. Agents split into three rows — Work & Research, Personal Life, System & Maintenance. Five newsletter and research agents in the first row, three personal agents in the second, five infrastructure agents in the third. It made the dashboard scannable in a way the flat grid never was. One glance tells you where energy is concentrated and where things have gone quiet.

VIII The Task Board March 25, 2026

Until now, getting an agent to do something meant typing a slash command and hoping for the best. There was no formal handoff — no brief, no scope, no review gate. I'd invoke /genos with a paragraph of instructions, and it would run until it was done or I killed it. For small tasks, that was fine. For anything ambitious, it was reckless.

So I built a task system. Not a to-do list — a proper work governance layer. Every task is a small PRD: a title, an objective, success criteria, scope boundaries, deliverables. Before an agent starts, the task goes through a brief. When it finishes, the work gets reviewed.

The part that matters is risk. Each task gets a risk level — low, medium, or high — and the review path changes accordingly. Low-risk work auto-completes. The agent finishes, the task closes, no one needs to look at it. Medium-risk tasks sit in review for forty-eight hours. If I don't weigh in, they auto-approve. High-risk work blocks until I explicitly sign off. Nothing ships unreviewed when the stakes are real.

The kanban board shows it all at once: five columns — In Progress, In Review, Planned, Completed, Cancelled — with risk badges and agent avatars on every card. At the top, an attention panel pins the medium and high-risk tasks that are waiting on me, sorted by how long they've been waiting. It's the "drop everything and look at this" surface I didn't know I needed.

But the feature I'm proudest of is the one most people wouldn't notice. It's called the Challenge Protocol. When an agent receives a task, the prompt doesn't just say "do this." It says: Before you begin, challenge this brief. Is the objective framed the right way? Are there gaps in the success criteria? Is the scope too broad or too narrow? If you see a better approach, say so — and update the task yourself.

The agents are explicitly invited to push back. They can rewrite success criteria, reframe the objective, suggest scope changes — all before they commit to the work. If the brief is solid, they say so and proceed. If it's not, they improve it first.

I didn't expect how much this would change the dynamic. Writing a task for Genos stopped feeling like filing a ticket and started feeling like briefing a colleague. Here's what I think we need. Tell me if I'm wrong. The fact that it sometimes does push back — "this scope is too broad, I'd split this into two deliverables" — makes the whole system feel less like automation and more like collaboration.

Each task also has a verification method. Some agents self-verify. Some get audited by Phoenix — the work-verification agent that cross-checks claims against actual code and file state. High-risk tasks require my explicit sign-off. The right amount of oversight for the right level of stakes.

IX The Shipping Calendar March 25, 2026

The task board shows what's happening now. But I kept wanting to look backwards — to see what had actually shipped, over time, across the whole roster. So a fifth page appeared: the shipping calendar.

Everything your agents shipped. March 2026 — 25 items across 16 active days and 10 agents. The orange squares are completions. The empty squares are weekends and quiet days.

A month view with completion markers. Click a day and you see what finished — which agent, which task, the risk level. A legend at the top lines up the avatars of every agent who shipped something that month. Summary stats: 25 items shipped, 16 active days, 10 agents involved.

The calendar answered a question I'd been carrying around without realizing it. When you work with agents every day, the individual sessions blur together. You remember the big moments — a successful literature review, a newsletter draft that needed no editing — but the steady accumulation of small completions is invisible. The calendar made it visible. You can see the rhythm: clusters of activity mid-week, quiet weekends, a burst when a deadline approaches.

There's something satisfying about watching a grid fill up. It's the same feeling as a GitHub contribution graph, except instead of commits, it's agent deliverables. Evidence that the system is doing real work, not just existing.

X Samuel Joins the Roster March 25, 2026

On March 25th, four days after the original build, I added myself to the dashboard.

Not as an admin panel or a settings icon. As an agent card. Same treatment as the others: pixel-art avatar, domain label ("Human Operator"), class badge. The label reads "C-Class." The species badge reads "Homo sapiens." Below the three agent rows — Work & Research, Personal Life, System & Maintenance — there's a fourth section. It's titled "Useless Human." Population: one.

I gave myself a mood system too. Five levels, same as the agents. But where their moods are driven by session counts and file dates, mine pulls from a quote database — hundreds of lines sorted into categories like "struggling," "wry," "peak," and "general." The dashboard picks one based on the current mood state and displays it next to my card in an italicized block. Every refresh, a different one.

The joke is the point. The agents have mythological names, literary references, serious domains. Putting myself in the same grid with "C-Class" and "Homo sapiens" is a reminder that the whole system exists to serve the one member of the roster who can't run on a cron job. It keeps the hierarchy honest. They have faces and voices and moods. So do I. And by most measures, they're more productive.

XI The Messenger Goes Live March 24, 2026

When I wrote section VI, the messenger was a mockup. A paper plane icon with a badge, a panel with placeholder threads. It looked right but did nothing.

It's real now. A database-backed message queue where agents post one-way notifications — not chat, just signals. Chiron flags when health check-ins are overdue. Borges reports on vault maintenance runs. Midas surfaces uncategorized transactions. Phoenix sends audit results. Each message carries the agent's avatar, a timestamp, and a read/unread state. The badge on the paper plane icon shows the total unread count. Open the panel, read the messages, the badge clears.

The messenger changed the dashboard from something I visit to something that visits me. I used to open it, scan the cards, and close it. Now there's a reason to notice — a small orange number that says someone has something to say. It's a subtle shift, but it means the agents don't just wait. They reach out. And because each message is deduplicated — the same content won't appear twice until you've read the first one — it never feels like spam. Just a patient tap on the shoulder.

The dashboard as it stands on March 26th. Fourteen agents in three lanes, each with a mood-driven left border — green for healthy, amber for needs attention, red for neglected. The human operator at the bottom, filed under "Useless Human."

XII

March 17th to March 25th. Nine days. The first agents were created on a Tuesday evening. The infrastructure was wired by Wednesday. Avatars on Friday. The dashboard built on Saturday night. Voices on Sunday morning. Moods by Sunday afternoon. The task board, calendar, and messenger landed the following Tuesday and Wednesday. I joined the roster myself on Wednesday evening.

Zero dependencies. Eighteen pixel-art portraits. A Bun server on port 3141 because the number felt right.

The part that sticks with me isn't any single feature. It's the accumulation. Faces made the agents recognizable. Voices gave them character. Moods made them dynamic. The task board gave them accountability. The calendar gave them a track record. The messenger gave them a way to reach me. Each layer was small on its own. Together, they turned a registry into a team.

I keep coming back to the Challenge Protocol. The idea that you hand an agent a brief and it's expected to push back if the brief is weak. That's the moment the system stopped being a tool and started feeling like a partnership. Tools don't have opinions about the quality of their instructions. Collaborators do.

The roster has eighteen entries now — fourteen active, four planned. The planned ones sit on the topology page as dashed outlines, waiting for their moment. Hermes for social media. Ripley for relationships. Kilgore for writing. Tron for infrastructure. They already have names. They'll get faces when they're ready.

Every time I open a modal and Saber hits me with a new quote, or Sisyphus looks more haggard than yesterday, or Chiron's border turns red because I haven't checked in — I remember why: tools should be fun, or you stop using them.

The bored hero and the father of psychology. 80 pixels each.
